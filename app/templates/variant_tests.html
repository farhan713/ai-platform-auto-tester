How the report reads

  • Per group — the original question's SQL is the baseline.
  • Each variant is marked ✓ match if its generated SQL equals the original's after normalizing (lowercase, collapsed whitespace, no trailing semicolon), else ✗.
  • You get a matched / total count per group and an overall consistency rate.
  • Full per-query detail (API calls, screenshots, validations) is on the normal run page — this report sits on top of it.

A group whose original failed / timed out (no baseline SQL) is flagged so you know the variants couldn't be compared.
